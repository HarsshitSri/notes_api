# Assets Plan — Notes API

Recommended documentation images based on the **current implementation** (7 REST endpoints, JWT auth, MySQL/H2, Docker Compose config, SpringDoc OpenAPI). No images are generated by this document — it defines what to capture and where to store it.

**Proposed root:** `assets/` (replace legacy `images/` after new assets are added)

**Accuracy checklist:** Before generating any diagram, verify against [`diagram-audit.md`](diagram-audit.md) and the live code under `note-app/src/main/java/com/Harshit/note_app/`. There is no booking feature — do not plan or create booking-flow assets.

```text
assets/
├── diagrams/
└── screenshots/
    ├── api/
    └── errors/
```

**Capture environment:** Run the app with `mvn spring-boot:run -Dspring-boot.run.profiles=h2` on port `8080` unless noted otherwise. Use Postman, Bruno, or curl with a visible client UI.

**Retire:** Existing files in `images/` are outdated (pre-JWT, wrong status codes and error shapes). Do not reuse; replace per mapping at the end.

### `assets/diagrams/request-flow-protected.png`

| Field | Detail |
| ----- | ------ |
| **Purpose** | Show the servlet/security filter path for authenticated note requests |
| **What should be visible** | Client → `SecurityFilterChain` → `JwtFilter` (runs on all routes; Bearer token optional on public paths) → authorization check → `NoteController` → `NoteService` → `SecurityUtils.getCurrentUser()` → `note.setUser(currentUser)` on create → `NoteRepository` → database. Include `401` path via `GlobalExceptionHandler.commence()` when token is missing on protected routes. `DELETE` returns `204 No Content`. |
| **When to capture** | Diagram from README "Servlet request flow" section — static, no runtime capture |
| **Used in** | README § Architecture Overview |

### `assets/diagrams/jwt-login-flow.png`

| Field | Detail |
| ----- | ------ |
| **Purpose** | Document how a JWT is obtained and used |
| **What should be visible** | Two swimlanes or sequences: **(1)** `POST /api/auth/login` → `UserService` → `AuthenticationManager` → `EmailPasswordAuthenticationProvider` → BCrypt check → `JwtService.generateToken()` → `UserLoginResponseDTO`. **(2)** Subsequent request with `Authorization: Bearer <token>` → `JwtFilter` → `SecurityContext` → protected endpoint. Note JWT subject = email. Registration (`POST /api/auth/register`) is a separate public path that returns `UserResponseDTO` only — **no JWT**. |
| **When to capture** | Static diagram from README login + protected flow sections |
| **Used in** | README § Authentication Overview, `Decisions.md` § JWT |

### `assets/diagrams/registration-flow.png`

| Field | Detail |
| ----- | ------ |
| **Purpose** | Show the public registration path (no JWT) |
| **What should be visible** | `POST /api/auth/register` → `AuthController` (`@Valid` → `400` on failure) → `UserService.register()` → duplicate checks → `UserMapper.toEntity()` → `PasswordEncoder` (BCrypt) → `UserRepository.save()` → `201 UserResponseDTO` (no token). Branch to `409` for duplicate username/email. |
| **When to capture** | Static diagram — no running app required |
| **Used in** | README § Architecture Overview |

### `assets/diagrams/database-schema.png`

| Field | Detail |
| ----- | ------ |
| **Purpose** | Document the JPA data model |
| **What should be visible** | Tables `users` (`id`, `username` UNIQUE, `email` UNIQUE, `password`, `created_at`) and `notes` (`id`, `title`, `content` TEXT, `user_id` FK NOT NULL, `created_at`, `updated_at`). Relationship: one `users` row to many `notes` via `notes.user_id` — **unidirectional** (no `notes` collection on `users`). Mark `password` as not exposed in API. Use snake_case for physical column names. |
| **When to capture** | Generate from `model/User.java` and `model/Note.java` — static ERD, no live DB required. Optionally verify against H2 console after `h2` profile startup |
| **Used in** | README § Database Overview |
